# Git plus Google Docs

Doctrine is like Git and Google Docs had a baby: a shared workspace that feels live and collaborative like Google Docs, but keeps the developer-native affordances of cloning, pushing, syncing, and working from the CLI.

The simple version:

- Like Google Docs, there is one shared cloud place where people can inspect and edit the same work.
- Like Git, you can clone it, push changes, sync workspaces, and treat the content as files instead of opaque app data.
- Better than both for agents, because the CLI can read files directly from the cloud without first pulling a local checkout.
- Better than Google Docs for structured work, because Doctrine supports many file types and stores them as regular formats such as JSON, CSV, Markdown, and other well-known standards.
- More agent-friendly and more human-friendly because humans, agents, scripts, and apps can all operate on the same underlying files through interfaces that match how they already work.

## Short Pitch

Doctrine is Google Docs for project files, with Git-style clone, push, and sync. It gives teams and agents a live cloud workspace made of normal file formats, readable from the CLI without pulling everything locally.

## Positioning Notes

The comparison should land as familiar first, then stronger:

- Google Docs explains the live shared workspace.
- Git explains clone, push, sync, history, and local workflows.
- Regular file formats explain openness and portability.
- CLI read-without-pull explains why this is especially useful for agents.

The core emotional point: Doctrine is not another document app or cloud drive. It is shared project context that both people and agents can actually work with.
