# S3

This guide covers setting up AWS S3 with LocalStack for local development, enabling a seamless transition to production.

## The Core Idea

LocalStack emulates AWS services locally. The key to seamless local-to-production parity is using a single environment variable—`AWS_ENDPOINT_URL`—to switch between LocalStack and real AWS:

- **Local**: Set `AWS_ENDPOINT_URL` to LocalStack's URL
- **Production**: Don't set `AWS_ENDPOINT_URL` (or leave it undefined)

The S3 client detects the endpoint and behaves accordingly.

## Environment Variables

Variable Local Production `AWS_REGION` `us-east-1` Your actual region `AWS_ACCESS_KEY_ID` `test` Real AWS key `AWS_SECRET_ACCESS_KEY` `test` Real AWS secret `AWS_ENDPOINT_URL` `http://localhost:<port>` *(not set)* `AWS_S3_BUCKET` `helloworld-files` Your bucket name

**Critical insight**: When `AWS_ENDPOINT_URL` is not set, the AWS SDK automatically uses the real AWS endpoints. This means the same code works locally and in production with zero changes—just different environment variables.

## S3 Client Setup

```
import { S3Client } from "@aws-sdk/client-s3";
import { env } from "./env";

let s3Client: S3Client | null = null;

export const getS3Client = (): S3Client => {
  if (s3Client) return s3Client;

  const config: ConstructorParameters<typeof S3Client>[0] = {
    region: env.AWS_REGION,
    credentials: {
      accessKeyId: env.AWS_ACCESS_KEY_ID,
      secretAccessKey: env.AWS_SECRET_ACCESS_KEY,
    },
  };

  if (env.AWS_ENDPOINT_URL) {
    config.endpoint = env.AWS_ENDPOINT_URL;
    config.forcePathStyle = true;
  }

  s3Client = new S3Client(config);
  return s3Client;
};
```

**Why `forcePathStyle: true`?**

AWS uses virtual-hosted-style URLs by default (`bucket.s3.region.amazonaws.com`), but LocalStack requires path-style URLs (`localhost:4566/bucket`). This option is only needed when `AWS_ENDPOINT_URL` is set.

## Auto-Creating Buckets Locally

In development, you may want buckets created automatically:

```
import {
  CreateBucketCommand,
  HeadBucketCommand,
} from "@aws-sdk/client-s3";

const ensureBucketExists = async (bucket: string): Promise<void> => {
  const isDev = env.AWS_ENDPOINT_URL || env.NODE_ENV !== "production";
  if (!isDev) return;

  const client = getS3Client();

  try {
    await client.send(new HeadBucketCommand({ Bucket: bucket }));
  } catch (error: unknown) {
    const err = error as {
      name?: string;
      $metadata?: { httpStatusCode?: number };
    };

    if (err.name === "NotFound" || err.$metadata?.httpStatusCode === 404) {
      await client.send(new CreateBucketCommand({ Bucket: bucket }));
    } else {
      throw error;
    }
  }
};
```

Call this before upload operations. In production, buckets should be pre-created via infrastructure-as-code.

## Basic Operations

### Upload

```
import { PutObjectCommand } from "@aws-sdk/client-s3";

export const uploadToS3 = async (
  buffer: Buffer,
  key: string,
  contentType: string,
): Promise<void> => {
  const client = getS3Client();
  await ensureBucketExists(env.AWS_S3_BUCKET);

  await client.send(
    new PutObjectCommand({
      Bucket: env.AWS_S3_BUCKET,
      Key: key,
      Body: buffer,
      ContentType: contentType,
    }),
  );
};
```

### Download

```
import { GetObjectCommand } from "@aws-sdk/client-s3";

export const downloadFromS3 = async (key: string): Promise<Buffer> => {
  const client = getS3Client();

  const response = await client.send(
    new GetObjectCommand({
      Bucket: env.AWS_S3_BUCKET,
      Key: key,
    }),
  );

  const stream = response.Body;
  const chunks: Uint8Array[] = [];
  for await (const chunk of stream as AsyncIterable<Uint8Array>) {
    chunks.push(chunk);
  }
  return Buffer.concat(chunks);
};
```

```
  user = aws_iam_user.s3_user.name

        Effect = "Allow"

          "s3:GetObject",
          "s3:PutObject",
          "s3:DeleteObject",
```

## Troubleshooting

### Connection Refused

LocalStack isn't running or port mapping is incorrect. Verify with:

```
curl http://localhost:4566/_localstack/health
```

### Invalid Bucket Name

LocalStack is stricter about bucket names than AWS in some versions. Use lowercase, no underscores.

### Path-Style vs Virtual-Hosted-Style Errors

Ensure `forcePathStyle: true` is set when using LocalStack. The AWS SDK defaults to virtual-hosted-style, which LocalStack doesn't support.

# S3

This guide covers setting up AWS S3 with LocalStack for local development, while keeping the same application code path for production AWS S3.

## Core Idea

Use one switch: `AWS_ENDPOINT_URL`.

- Local: set `AWS_ENDPOINT_URL` to the LocalStack gateway URL.
- Production: do not set `AWS_ENDPOINT_URL`; the AWS SDK will use real AWS endpoints.

When `AWS_ENDPOINT_URL` is set, the S3 client must use `forcePathStyle: true` because LocalStack uses path-style bucket URLs such as `localhost:4566/bucket/key`.

## Environment Variables

Application services that access S3 usually need:

Variable Services Secret Notes `AWS_REGION` backend, worker No Defaults to `us-east-1` `AWS_ACCESS_KEY_ID` backend, worker Yes `test` locally, real key in prod `AWS_SECRET_ACCESS_KEY` backend, worker Yes `test` locally, real key in prod `AWS_ENDPOINT_URL` backend, worker No LocalStack URL locally; unset in prod `AWS_S3_BUCKET` backend, worker No Bucket name

LocalStack itself should use a separate env override file instead of inline service env vars:

```
# .env.localstack
SERVICES=s3
DEBUG=0
DATA_DIR=/data
LS_LOG=warn
AWS_DEFAULT_REGION=us-east-1
```

Application local env example:

```
# .env.local
LOCALSTACK_PORT=4566
AWS_REGION=us-east-1
AWS_ACCESS_KEY_ID=test
AWS_SECRET_ACCESS_KEY=test

```

## Zap LocalStack Configuration

Use a dedicated env file stack for the LocalStack container:

```
docker:
  localstack:
    image: localstack/localstack:latest
    ports:
      - "${LOCALSTACK_PORT}:4566"
    env:
      - .env.localstack
    volumes:
      - localstack-data:/data
```

Key settings:

- `SERVICES=s3` starts only S3, which is faster and less noisy.
- `DATA_DIR=/data` plus a Docker volume preserves LocalStack data between restarts.
- Keep LocalStack container env separate from app env so app secrets are not passed into the LocalStack container.

## Auto-Creating Buckets Locally

In development, buckets can be created automatically. In production, buckets should be pre-created by infrastructure.

```
import {
  CreateBucketCommand,
  HeadBucketCommand,
} from "@aws-sdk/client-s3";

const shouldBootstrapBucket = () =>
  Boolean(env.AWS_ENDPOINT_URL) || env.APP_ENV === "development";

const ensureBucketExists = async (bucket: string): Promise<void> => {
  if (!shouldBootstrapBucket()) return;

  const client = getS3Client();

  try {
    await client.send(new HeadBucketCommand({ Bucket: bucket }));
  } catch (error: unknown) {
    const err = error as {
      name?: string;
      $metadata?: { httpStatusCode?: number };
    };

    if (err.name === "NotFound" || err.$metadata?.httpStatusCode === 404) {
      await client.send(new CreateBucketCommand({ Bucket: bucket }));
    } else {
      throw error;
    }
  }
};
```

## Browser Uploads And CORS

If the browser uploads directly to S3 with presigned PUT URLs, the bucket needs CORS. This is true for LocalStack too because the browser still enforces CORS for `localhost` origins.

For local development, apply CORS as part of the bucket bootstrap when `AWS_ENDPOINT_URL` is set:

```
import { PutBucketCorsCommand } from "@aws-sdk/client-s3";

const ensureLocalBucketCors = async (bucket: string): Promise<void> => {
  if (!env.AWS_ENDPOINT_URL) return;

  await getS3Client().send(
    new PutBucketCorsCommand({
      Bucket: bucket,
      CORSConfiguration: {
        CORSRules: [
          {
            AllowedHeaders: ["*"],
            AllowedMethods: ["GET", "HEAD", "PUT"],
            AllowedOrigins: [
              env.FRONTEND_URL,
              "http://localhost:*",
              "http://127.0.0.1:*",
            ],
            ExposeHeaders: ["ETag"],
            MaxAgeSeconds: 300,
          },
        ],
      },
    }),
  );
};
```

For production, configure CORS on the real S3 bucket with the exact frontend origin.

### Presigned Browser Upload

Install the presigner when the browser needs direct S3 upload/download URLs:

```
pnpm add @aws-sdk/client-s3 @aws-sdk/s3-request-presigner
```

```
import { GetObjectCommand, PutObjectCommand } from "@aws-sdk/client-s3";
import { getSignedUrl } from "@aws-sdk/s3-request-presigner";

export const createPresignedS3PutUrl = async (input: {
  contentType: string;
  key: string;
}) => {
  await ensureBucketExists(env.AWS_S3_BUCKET);

  return getSignedUrl(
    getS3Client(),
    new PutObjectCommand({
      Bucket: env.AWS_S3_BUCKET,
      Key: input.key,
      ContentType: input.contentType,
    }),
    { expiresIn: 15 * 60 },
  );
};

export const createPresignedS3GetUrl = async (key: string) => {
  await ensureBucketExists(env.AWS_S3_BUCKET);

  return getSignedUrl(
    getS3Client(),
    new GetObjectCommand({
      Bucket: env.AWS_S3_BUCKET,
      Key: key,
    }),
    { expiresIn: 60 * 60 },
  );
};
```

## Checklist

### Local Development

1. Add LocalStack to `zap.yaml`.
2. Create `.env.localstack` and reference it from the LocalStack service.
3. Set app S3 variables in `.env.local`.
4. Enable `forcePathStyle` when `AWS_ENDPOINT_URL` is set.
5. Auto-create buckets in dev/local mode.
6. If using direct browser uploads, apply local bucket CORS during bucket bootstrap.

### Production

1. Create the private S3 bucket with public access blocked.
2. Add bucket CORS if browsers upload directly to S3.
3. Configure AWS credentials and bucket name in the deployment secret/env store.
4. Leave `AWS_ENDPOINT_URL` unset.

## Troubleshooting

### Connection Refused

LocalStack is not running or the port mapping is wrong:

```
curl http://localhost:4566/_localstack/health
```

### Browser Upload Fails CORS Preflight

The bucket needs CORS even when using LocalStack. Verify the preflight returns `Access-Control-Allow-Origin` for the frontend origin.

### Deepgram Or External Services Cannot Fetch LocalStack URLs

A presigned LocalStack URL still points at `localhost`, so external services cannot fetch it. Use real S3 or a public tunnel/public endpoint for end-to-end local tests that require third-party fetches.

### Path-Style vs Virtual-Hosted-Style Errors

Set `forcePathStyle: true` whenever `AWS_ENDPOINT_URL` is set.
