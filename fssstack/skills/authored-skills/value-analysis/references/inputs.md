Request only the missing information needed to build the documentation library.

Minimum request when starting from little context:

- target folder for the documentation library
- product name
- one-sentence product description
- primary users or buyers
- source material the user can provide

Useful source material:

- source code
- product docs
- screenshots or designs
- landing page
- pitch deck
- product story
- value proposition statement
- testimonials
- sales notes
- support tickets
- competitor names or links
- customer calls or research notes

Prefer source code, product docs, live product screenshots, and real customer language.

Use weaker inputs when stronger inputs are unavailable. Mark conclusions as inference when the input does not directly support them.

Do not ask for every possible input at once when the user has already provided enough to start.
